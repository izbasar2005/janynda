# Janynda - Платформа психологической поддержки

Janynda -- платформа для психологической поддержки людей с серьёзными заболеваниями (онкология и другие). Система объединяет пациентов в группы по диагнозу, предоставляет личный дневник с ИИ-мониторингом психологического состояния и обеспечивает профессиональную помощь психологов при критических ситуациях.

## Участники системы

| Роль | Описание | Доступ |
|------|----------|--------|
| **Пациент** (patient) | Человек, проходящий лечение | Дневник, группы, личные чаты, запись к врачу |
| **Волонтёр** (volunteer) | Поддержка и модерация групп | Группы, личные чаты, дневник |
| **Доктор** (doctor) | Медицинский специалист | Кабинет врача, приёмы, диагнозы, группы (свои) |
| **Терапевт** (doctor + is_therapist) | Врач общей практики | Все группы, все пациенты, направления, записи |
| **Психолог** (psychologist) | Анализ сложных случаев | Кейсы ИИ, анонимные/полные данные |
| **Админ** (admin) | Управление пользователями | Управление ролями, врачами, новостями |
| **Суперадмин** (super_admin) | Полный доступ | Всё + статистика, дашборд |

## Функциональные модули

### 1. Группы поддержки (Топтар)

Групповые чаты по диагнозам. Участники: пациенты с похожим диагнозом, волонтёры, доктора.

- Создание и управление группами (доктор/админ/терапевт)
- Сообщения в реальном времени (WebSocket)
- Системные сообщения при добавлении/удалении участников
- Личные чаты между участниками группы
- Статусы прочтения сообщений

### 2. Личный дневник (Күнделік)

Приватное пространство пациента для записи мыслей и эмоций.

- Выбор настроения с визуальными эмодзи
- Календарь с отметками дней записей (стрик-трекер)
- Динамическое приветствие по времени суток
- ИИ-анализ каждой записи (скрыт от пациента)

### 3. Запись к врачу (Дәрігерге жазылу)

- Просмотр списка врачей с фильтрацией
- Выбор свободного слота по дате/времени
- Уведомления о предстоящих приёмах (15 и 5 минут)
- Возможность отмены (за 30+ минут до приёма)

### 4. Кабинет врача (Дәрігер кабинеті)

- Список пациентов и приёмов
- Установка диагноза и клинических заметок
- Управление статусами приёмов (одобрено/завершено)
- Терапевт: направления к специалистам, добавление в группы

### 5. Направления (Бағыттар)

Терапевт может:
- Поставить диагноз (сохраняется в профиле пациента)
- Выписать направление к специалисту
- Сразу записать пациента к конкретному врачу
- Добавить пациента в группу поддержки по диагнозу

### 6. Психологические кейсы

Автоматически создаются при ИИ-оценке в жёлтой (60-79) или красной (0-59) зоне.

- Жёлтая зона: психолог видит анонимный текст
- Красная зона: полный доступ к данным пациента (только назначенному психологу)
- Оценка психолога: подтвердить/опровергнуть ИИ
- Эскалация или закрытие кейса

### 7. Уведомления

- Push-уведомления о приёмах
- Напоминания за 15 и 5 минут
- Выбор модальности (видеозвонок, текст, отмена)

### 8. Новости

- Публикация статей и рекомендаций
- Избранная новость на главной странице
- CRUD для админов

## ИИ-система мониторинга

### Принцип работы

Каждая запись в дневнике автоматически анализируется через Anthropic Claude API. ИИ видит только текст без личных данных.

### Система оценок (0-100)

| Зона | Баллы | Действие |
|------|-------|----------|
| Зелёная | 80-100 | Мониторинг продолжается, действий нет |
| Жёлтая | 60-79 | Анонимный кейс для психолога |
| Красная | 0-59 | Полный кейс с доступом к данным пациента |

### Матрица доступа к данным ИИ

| Данные | Пациент | Волонтёр | Врач | Психолог | Админ | Суперадмин |
|--------|---------|----------|------|----------|-------|------------|
| Свой AI-балл | нет | нет | нет | да | да | да |
| Анонимный текст (жёлтая) | нет | нет | нет | да | нет | да |
| Полные данные (красная) | нет | нет | нет | назначенный | да | да |

### Фоновый воркер

`RunDiaryAiRetryWorker` каждую минуту повторяет неудавшиеся ИИ-анализы и создаёт PsychCase при обнаружении жёлтой/красной зоны.

## Запуск

### Требования

- Docker и Docker Compose
- (Опционально) Go 1.22+ и Node.js 20+ для локальной разработки

### Через Docker Compose (рекомендуется)

```bash
# Клонировать проект
git clone <repository-url>
cd janynda

# Создать .env или задать ключ в docker-compose.dev.yml
# ANTHROPIC_API_KEY=sk-ant-...

# Запустить
docker compose -f docker-compose.dev.yml up --build
```

Сервисы:
- **Frontend**: http://localhost:5173
- **API**: http://localhost:8080
- **PostgreSQL**: localhost:5432

### Переменные окружения

| Переменная | Описание | По умолчанию |
|-----------|----------|-------------|
| `DATABASE_URL` | Строка подключения PostgreSQL | `postgres://postgres:postgres@db:5432/janymda?sslmode=disable` |
| `JWT_SECRET` | Секрет для подписи JWT | `dev_secret_change_me` |
| `ANTHROPIC_API_KEY` | Ключ API Anthropic | — |
| `ANTHROPIC_MODEL` | Модель Claude | `claude-haiku-4-5` |
| `AUTO_SEED` | Автозаполнение демо-данными | `true` |
| `PORT` | Порт API | `8080` |
